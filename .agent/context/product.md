# Product Context

## Product name
Itqan

## Core problem
Users need a structured, calm, long-term system for Quran memorization and revision.
Most apps either behave like generic readers, trackers, or content browsers rather than a memorization workflow system.

## Primary user outcomes
The user should be able to:
- know what to do today
- continue from where they stopped
- memorize with low friction
- review due material reliably
- self-test with reduced assistance
- recover after missed days without guilt-heavy UX

## Primary app areas
1. Today
2. Plan
3. Session
4. Progress
5. Library

## Product promises
- Arabic-first usability
- calm and focused interaction
- meaningful progress
- respectful recovery
- no unnecessary complexity
- strong continuity over time

## V1 target
Deliver a production-minded MVP with:
- Today
- Plan overview
- Session modes
- Progress summary
- Library basics
- essential settings
- strong design-token based UI foundation

## Non-goals for V1
- social features
- broad community features
- heavy analytics dashboards
- generic content-platform behavior
- ornamental visual exploration
- premature backend complexity unless clearly needed

## UX principles
- today-first
- recognition over recall
- progressive disclosure
- review is first-class
- one dominant action per primary screen
- low cognitive load
- clear system status
