# Skill — UI Parity Audit

## Purpose
Check Flutter UI against the design source of truth.

## Audit checklist
- tokens used correctly
- typography roles used correctly
- product components preferred over raw generic widgets
- primary CTA hierarchy correct
- RTL behavior correct
- Quran content styling correct
- review importance not visually downgraded
- no dashboard-heavy drift on core screens

## Output
List:
- parity matches
- deviations
- necessary fixes
- acceptable temporary gaps
