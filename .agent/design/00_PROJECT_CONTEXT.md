# Project Context — Itqan

## Product identity
Itqan is an Arabic-first Quran memorization and revision app.
It is **not** a generic Quran reader.
It is a long-term memorization operating system that helps users:
- memorize new verses
- review previously memorized material
- test themselves
- continue for months and years until strong retention and mastery

## Product tone
The app should feel:
- calm
- trustworthy
- focused
- supportive
- disciplined
- modern but not trendy

Avoid:
- decorative religious overload
- loud gamification
- dashboard-heavy screens
- shallow productivity-app aesthetics

## Primary product areas
1. Today
2. Plan
3. Session
4. Progress
5. Library

## Core UX posture
The app should answer:
- What should I do now?
- What is due today?
- Where did I stop?
- Am I on track?

## Why this matters for implementation
The design system must serve the memorization workflow, not generic app UI.
The agent must translate design foundations into product-specific Flutter components, not just theme tokens.
